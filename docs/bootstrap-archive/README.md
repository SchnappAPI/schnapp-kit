# `.kit-bootstrap/` — Temporary Bootstrap Payload for `schnappapi/schnapp-kit`

This directory is **temporary**. Its sole purpose is to ferry the plan and execution instructions for creating the new `schnappapi/schnapp-kit` repository through two sessions that can't both reach all the necessary services from one place.

## The handoff chain

```
This session (claude.ai, scoped to schnapp-bet only)
    │
    │ wrote these files
    ▼
.kit-bootstrap/ in schnapp-bet @ branch claude/awesome-cori-p23yR
    │
    │ user pastes HANDOFF-A.md into ↓
    ▼
Claude chat (claude.ai) session that CAN create GitHub repos
    │  - creates schnappapi/schnapp-kit (public)
    │  - writes HANDOFF-B-claude-code.md to .kit-bootstrap/ (customized
    │    from HANDOFF-B-template.md with the new repo's clone URL)
    │  - commits HANDOFF-B-claude-code.md to schnapp-bet
    ▼
User pastes HANDOFF-B-claude-code.md into ↓
    ▼
Claude Code session on Schnapps-MBP (Mac, full local access)
    │  - clones schnapp-kit
    │  - reads .kit-bootstrap/00–06 from schnapp-bet
    │  - executes the build (fork ECC, vendor mattpocock/skills,
    │    overlay schnapp-bet promotions, language packs, scripts, skills)
    │  - commits and pushes schnapp-kit
    │  - copies .kit-bootstrap/ → schnapp-kit/docs/bootstrap-archive/
    │    (for posterity)
    │  - deletes .kit-bootstrap/ from schnapp-bet, commits, pushes
```

After the Mac session finishes, this directory **must not exist in schnapp-bet anymore**. The cleanup is part of HANDOFF-B's responsibilities and is documented in `06-cleanup.md`.

## v1 is the start, not the finish

The artifacts produced by this bootstrap are an opinionated **v1**. They are deliberately not the final state. The kit ships with two evolution skills (`/discover-kit-additions`, `/audit-against-kit`) that expect to be invoked repeatedly over time as more upstream repos and project repos are mined for content. Expect `core/`, `vendored/`, `overlays/`, and `language-packs/` to grow significantly after this initial build.
